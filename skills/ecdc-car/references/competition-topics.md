 ## 比赛现场策略
 
 ### 赛前准备清单
 
 1. **硬件检查**:
    - 所有传感器功能测试 (逐个例程跑一遍)
    - 电机编码器方向确认 (正转时计数增加的通道标记)
    - 电池电量充足 (带 2 组备用)
    - K230 固件烧录 + LAB 校准确认
 
 2. **代码备份**:
    - 每个模块的可用基���代码 (独立工程)
    - 常用 PID 参数的三套预设 (快速/均衡/稳定)
    - 各传感器裸读代码 (用于现场排查)
 
 3. **工具清单**:
    - 串口调试助手 + USB-TTL 模块
    - 万用表 (测电压/短路)
    - 示波器 (测 PWM/编码器波形, 有的话)
    - 笔记本 + Keil + K230 CanMV IDE
 
 ### 选题策略
 
 1. 先通读所有题目, 找出与自己模块匹配度最高的
 2. 优先选自己已有成熟代码的题型 (巡线/色块/测距)
 3. 避开需额外采购模块的题目
 4. 控制题核心是"稳"不是"快"——宁可慢一点, 不能翻车
 
 ### 现场调试节奏
 
 ```
 第1小时: 选题 + 确认引脚分配无冲突
 第2-4小时: 搭建基础框架 (电机+编码器+巡线/视觉 基本跑通)
 第5-12小时: 算法调优 (PID 参数, 巡线逻辑, 通信稳定)
 第13-20小时: 完整流程测试 + 边界条件处理
 最后4小时: 代码固化 + 电池更换 + 展示准备
 ```
 
 ### 常见现场问题
 
 | 问题 | 排查方向 |
 |------|---------|
 | 电机不转 | PWM 启动 + 方向引脚 + 电池电压 |
 | 巡线跑偏 | PID 参数 + 传感器高度 + 光线变化 |
 | K230 无数据 | 共地 + 波特率 + K230 侧程序是否运行 |
 | 编码器计数不准 | 中断优先级 + 线缆屏蔽 + 电源噪声 |
 | 小车抖/振荡 | PID 增益过大 + 机械松动 |
